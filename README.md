# 🏨 Hotel Budget Management | UI/UX Case Study

A UI/UX case study for a **Hotel Budget & Financial Management System** that enables hotel owners and managers to monitor budgets, expenses, revenue, profit, and loss from a centralized dashboard. The goal of this project is to replace manual spreadsheet-based financial tracking with a modern web and mobile solution that provides real-time business insights and automated notifications.

---

## 📌 Project Overview

This project is a UI/UX design case study for a Hotel Budget and Financial Management System designed to help hotel owners and managers monitor budgets, expenses, revenue, profit, and loss from a single platform. The solution aims to replace manual Excel-based tracking with a centralized web and mobile application that provides real-time financial visibility and notifications.

The system supports two user roles:
Hotel Owner / Admin (CEO, COO, Owner)
Hotel Manager

For this assignment, the focus was on designing and prototyping the features related to financial monitoring, business performance tracking, and automated notifications

---

## 🚨 Problem Statement

The organization manages more than 50 hotels, each operating with a monthly budget approved by senior management. Currently, budgets and expenses are tracked manually using spreadsheets, making it difficult to monitor budget utilization, profitability, and financial performance in real time.
This often leads to:
Delayed awareness of budget overruns
Missed profit opportunities
Lack of timely communication between hotel managers and leadership
Manual reporting errors
The goal was to create a centralized platform that provides clear financial visibility and automated alerts for both hotel managers and owners.

## 🎨 Design Process

The design process started with understanding the full problem statement and breaking it down into clear user needs. I first studied how hotels currently manage budgets, expenses, and financial reporting using manual Excel sheets, and identified the main issues such as delayed updates, lack of real-time visibility, and difficulty in tracking profit and loss across multiple hotels. After this, I reviewed all the user stories and grouped them based on two main roles: Hotel Owner (Admin) and Hotel Manager. This helped me clearly define what each user should see and do in the system.

Once the requirements were clear, I planned the structure of the product by creating an information flow for the dashboard and notification system. I focused on how users would move from one screen to another and how financial data like budget, expenses, revenue, and profit should be displayed in a simple and easy-to-understand way. After that, I started designing wireframes in Figma to decide the layout of each screen, including dashboards, financial summary pages, and notification panels.

After finalizing the wireframes, I improved the design by adding proper spacing, consistent components, and a clean layout using Figma features like auto layout and reusable components. I also made sure that important information such as budget alerts and profit notifications was clearly visible and easy to understand. Finally, I created an interactive prototype to show how the system would work in real usage, including navigation between pages and interaction with key features like viewing financial summaries and receiving alerts. This process helped ensure that the final design was user-friendly, structured, and aligned with real business needs.
